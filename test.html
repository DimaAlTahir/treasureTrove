Testing Efforts

User Acceptance Test Plan

  1. The app will be tested by Antreas Messios and reported by Dima AlTahir
  2. Every aspect implemented in the app, from index page to leader board will be tested
  3. Testing will start when app is close to being finished. ie: when the leader board API is implemented
  4. The results will be recorded on the User Acceptance test page linked on the landing page of the app
  5. Bugs discovered will be noted by the tester(Antreas Messios) and reported by Dima AlTahir in a group meeting, to discuss best possible approach to fixing the the different bugs in each aspect.
  6. once the bugs are reported and discussed, party responsible for implementing said aspect will fix its bugs using the method agreed upon by the group.
  7. Retesting will occur once the 1st batch of bugs found is fixed.
