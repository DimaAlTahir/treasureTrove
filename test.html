Testing and Evaluation Efforts

Nielsenâ€™s 10 Heuristics Compliance Assessment

  1. Visibility of system status
    1. Messages indicating user whereabouts are available throughout all app pages.
    2. Messages indicating user score and progress are available where needed.
    3. Loading messages are available where needed.
    4. Messages indicating courses of action available for user exist throughout all pages.
    5. Buttons change color when they are being pressed.
    6. Links change color once they have been used.
  2. Match between system and the real world
    1. Everyday language used to communicate messages to user; avoided using technical language.
    2. Icons used to indicate usage of camera in QR scanner display.
  3. User control and freedom
    1. Courses of action available to user are indicated using various messages and buttons throughout all app pages.
      1. User can start a new game then choose to continue it.
      2. If user wants to start a new game, a message is shown to tell them how.
      3. User can skip questions they choose not to answer using a skip button.
      4. User can activate QR code scanner using two buttons; one to show the scanner and one to switch the camera used.
  4. Consistency and standards
    1. All app pages follow the same template.
    2. App design follows platform consistency standards.
  5. Error prevention
    1. Types of input used are consistent with desired answered to prevent the user from accidentally entering wrong answer.
    2. A message is displayed to alert user if user enters a name that has been used before.
    3. A message is displayed to alert user if user device lacks a feature needed in the app.
  6. Recognition rather than recall
    1. All app pages follow the same template; user does not need to guess what each aspect means.
    2. Buttons are descriptive of their use; no recall needed on user's part.
    3. Icons compatible with real world design; no recall needed on user's part.
  7. Flexibility and efficiency of use
    1. App can be used by both experienced and non-experienced users.
  8. Aesthetic and minimalist design
    1. App interface shows only essential parts.
    2. Wording in interface is brief and descriptive.
    3. Icons compatible with real world design; no recall needed on user's part.
  9. Help users recognize, diagnose, and recover from errors
  10. Help and documentation

User Acceptance Test Plan

  1. The app will be tested by Antreas Messios and reported by Dima AlTahir
  2. Every aspect implemented in the app, from index page to leader board will be tested
  3. Testing will start when app is close to being finished. ie: when the leader board API is implemented
  4. The results will be recorded on the User Acceptance test page linked on the landing page of the app
  5. Bugs discovered will be noted by the tester(Antreas Messios) and reported by Dima AlTahir in a group meeting, to discuss best possible approach to fixing the the different bugs in each aspect.
  6. Once the bugs are reported and discussed, party responsible for implementing said aspect will fix its bugs using the method agreed upon by the group.
  7. Retesting will occur once the 1st batch of bugs found is fixed.
